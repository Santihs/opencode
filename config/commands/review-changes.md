# /review-changes

Review current worktree changes.

## What It Reviews

1. **Staged changes** — what's ready to commit
2. **Unstaged changes** — working on but not staged
3. **Untracked files** — new files not yet tracked

## Focus Areas

- Code quality issues
- Security concerns (secrets, keys)
- Missing tests
- Breaking changes
- Inconsistent patterns

## Output

- Summary of issues by severity
- Specific file:line references
- Suggestions for improvement
