# Ask Questions If Underspecified

## When to Use

Use this skill when:

- Requirements are unclear or ambiguous
- There are multiple valid interpretations
- You're unsure what the user wants
- The task could have unintended consequences
- Important details are missing

## The Principle

**It's faster to ask than to fix mistakes.**

## What to Do

1. **Pause** — don't assume what the user wants
2. **Identify** — what questions need answers?
3. **Ask** — use the question tool with clear options
4. **Confirm** — repeat back understanding before implementing

## Good Questions

- "Which approach would you prefer: A or B?"
- "Should I also update related tests?"
- "Do you want the commit message to include X?"
- "Should I run verification checks afterward?"

## Bad Questions

- "I'll just implement it this way" (assumes)
- "This seems obvious, I'll do it" (doesn't clarify)
